# نظام البث المباشر اليومي التلقائي على يوتيوب — دليل الإعداد الكامل

نظامٌ يعمل على خادم سحابي مجاني على مدار الساعة، ويبثّ **كل يوم قناةً واحدة** بالتناوب:
فيديو **عشوائي** يُسحب من قناة اليوم، ويُنشأ له **بثّ مباشر جديد بعنوانه الخاص** عبر YouTube API.

**دورة التناوب (أربعة أيام):** رمال النجاة ← Sands of Survival ← أصوات الجريمة ← Voices of Crime ← تُعاد.

---

## ملفات المشروع

| الملف | الوظيفة |
|---|---|
| `daily_live.py` | النظام الرئيسي (يُشغَّل يوميًا) |
| `authorize.py` | تفويض كل قناة والحصول على مفتاح التحديث |
| `notify.py` | إرسال إشعارات تيليجرام (بدء / انتهاء / خطأ) |
| `config.example.json` | نموذج الإعدادات (انسخه إلى `config.json`) |
| `run_daily.sh` | غلاف يشغّله cron |
| `requirements.txt` | مكتبات بايثون المطلوبة |

---

## ١) متطلبات يوتيوب (لكل قناة من الأربع)

- فعّل قناتك برقم الهاتف من `youtube.com/verify`.
- يجب ألّا يكون على القناة أي قيود خلال آخر 90 يومًا.
- فعّل البث المباشر من داخل القناة، وانتظر حتى 24 ساعة أول مرة قبل أن يعمل.
- العمر 16 سنة فأكثر.

⚠️ ابثّ فقط محتواك أنت. بما أن القنوات الأربع ملكك، فلا مشكلة في حقوق النشر.

---

## ٢) تجهيز الخادم السحابي المجاني

الخيار الموصى به: **Oracle Cloud Always Free** — أنشئ جهازًا افتراضيًا من نوع **Ampere (ARM)** بنظام **Ubuntu**، فهو مجاني دائمًا وقوي بما يكفي لبثّ واحد يوميًا.

بعد الدخول إلى الخادم عبر SSH، ثبّت المتطلبات:

```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv ffmpeg
mkdir -p ~/ytlive/work && cd ~/ytlive
# ارفع ملفات المشروع إلى هذا المجلد ثم:
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
pip install -U yt-dlp
```

> `yt-dlp` يتغيّر كثيرًا، فحدّثه دوريًا: `pip install -U yt-dlp`

---

## ٣) مشروع Google Cloud وواجهة YouTube API

1. ادخل إلى `console.cloud.google.com` وأنشئ **مشروعًا جديدًا**.
2. من **APIs & Services ← Library** فعّل **YouTube Data API v3**.
3. من **OAuth consent screen**: اختر **External**، واملأ البيانات الأساسية.
   - **مهم جدًا:** اضبط حالة النشر على **In production** (وليس Testing)، وإلا **تنتهي صلاحية مفاتيح التحديث بعد 7 أيام** ويتوقف النظام أسبوعيًا. قد تظهر شاشة "تطبيق غير موثّق" — اضغط **Advanced ← المتابعة**، فهذا طبيعي لاستخدامك الشخصي.
4. من **Credentials ← Create Credentials ← OAuth client ID**: اختر نوع **Desktop app**، ثم نزّل ملف الاعتماد وسمِّه `client_secret.json` وضعه في مجلد المشروع.

## ٤) تفويض القنوات الأربع (مرة واحدة لكل قناة)

يُنفَّذ على **جهازك الشخصي** (فيه متصفح)، لأن التفويض يحتاج تسجيل دخول:

```bash
python authorize.py client_secret.json token_channel1.json
python authorize.py client_secret.json token_channel2.json
python authorize.py client_secret.json token_channel3.json
python authorize.py client_secret.json token_channel4.json
```

في كل مرة: سجّل الدخول بالحساب الذي يملك القناة، واختر **القناة الصحيحة** عند السؤال. ثم انسخ ملفات `token_channelX.json` الأربعة و`client_secret.json` إلى مجلد المشروع على الخادم.

---

## ٥) ملف الكوكيز (للسحب التلقائي)

السحب التلقائي من يوتيوب على خادم سحابي يحتاج غالبًا ملف كوكيز لتسجيل الدخول:

1. في متصفحك، ثبّت إضافة تصدير الكوكيز (مثل "Get cookies.txt LOCALLY").
2. افتح `youtube.com` وأنت مسجّل الدخول، وصدّر الكوكيز بصيغة **Netscape**.
3. احفظ الملف باسم `cookies.txt` في مجلد المشروع على الخادم.

> الكوكيز تنتهي صلاحيتها مع الوقت. إن توقّف السحب فجأة، أعِد تصدير `cookies.txt`.
> هذه أضعف نقطة في النظام (كما اتفقنا)، لأن يوتيوب قد يطلب تحققًا من عناوين الخوادم السحابية.

## ٦) الإعدادات (config.json)

انسخ النموذج وعدّله:

```bash
cp config.example.json config.json
nano config.json
```

- ضع رابط كل قناة الصحيح في `youtube_channel_url` (يفضَّل رابط تبويب الفيديوهات، مثل `.../@handle/videos`).
- `epoch_date`: تاريخ بداية الدورة (يحدّد أي قناة تبدأ اليوم الأول).
- `work_dir`: مجلد التنزيل المؤقت (اتركه كما هو غالبًا).
- `reencode`: اتركه `true` للتوافق الأضمن، أو `false` (أخف على المعالج) إن كانت الفيديوهات جاهزة أصلًا.

## ٦.١) إشعارات تيليجرام

يرسل النظام رسالة عند **بدء البث** و**انتهائه** و**عند أي خطأ**.

1. افتح تيليجرام وابحث عن **@BotFather**، وأرسل `/newbot` واتبع الخطوات لإنشاء بوت. ستحصل على **توكن البوت** (Bot Token).
2. أرسل أي رسالة لبوتك أنت (حتى يستطيع مراسلتك).
3. للحصول على **chat_id**: ابحث عن **@userinfobot** وأرسل له `/start`، سيعطيك رقم معرّفك (chat_id). أو أنشئ مجموعة، أضِف بوتك إليها، واستخدم معرّف المجموعة.
4. ضع القيمتين في `config.json` داخل قسم `telegram`، واجعل `enabled` بقيمة `true`.

> لإرسال الإشعار لأكثر من جهة، اجعل `chat_id` قائمة، مثل: `["111", "-100222"]`.
> لإيقاف الإشعارات، اجعل `enabled` بقيمة `false`.

## ٧) التجربة اليدوية

```bash
cd ~/ytlive && source venv/bin/activate
python3 daily_live.py
```

راقب في YouTube Studio ظهور البث وتحوّله إلى "مباشر". إذا نجح، انتقل للجدولة.

---

## ٨) الجدولة اليومية (cron)

اجعل الغلاف قابلاً للتنفيذ:

```bash
chmod +x run_daily.sh
```

ثم `crontab -e` وأضِف (مثال: كل يوم الساعة 8 مساءً بتوقيت الخادم):

```
0 20 * * * /home/ubuntu/ytlive/run_daily.sh
```

> تحقّق من توقيت الخادم بـ `date`. لضبط المنطقة الزمنية:
> `sudo timedatectl set-timezone Asia/Riyadh`

## ٩) ملاحظات وصيانة

- **الحصص (Quota):** إنشاء بثّ واحد يوميًا ضِمن الحصة المجانية (10,000 وحدة/يوم) بلا قلق.
- **حدّث yt-dlp** كل فترة: `pip install -U yt-dlp`.
- **جدّد cookies.txt** إذا فشل السحب.
- **مدة البث** = مدة الفيديو المختار. يتوقف البث تلقائيًا عند انتهائه (enableAutoStop).
- راجع `daily.log` لأي أخطاء.

## استكشاف الأخطاء السريع

| العطل | الحل |
|---|---|
| فشل السحب / "Sign in to confirm" | جدّد `cookies.txt` من حساب مسجّل الدخول |
| توقف أسبوعيًا | حالة OAuth ليست "In production" → غيّرها |
| البث لا يتحوّل لمباشر | تأكد أن FFmpeg يرسل فعلًا، وأن البث المباشر مفعّل على القناة |
| خطأ ترميز/توافق | اجعل `reencode` = `true` |
| ضعف المعالج على الخادم | استخدم جهاز Ampere ARM، أو `reencode` = `false` مع فيديو جاهز |

## ترقيات ممكنة لاحقًا (أخبرني إن أردت)

- إشعار عبر تيليجرام/بريد عند بدء البث أو عند أي خطأ.
